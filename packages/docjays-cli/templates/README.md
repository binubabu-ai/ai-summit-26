# DocJays - Documentation Manager

This `.docjays/` folder contains all documentation, feature specifications, and AI context for this project. It's automatically managed by the DocJays CLI and is git-ignored by default.

- **sources/** - Cloned documentation from configured sources

- **context/** - AI context files (architecture, conventions, etc.)
- **cache/** - Cached data and embeddings
- **logs/** - Operation logs

## üîß Quick Start

### View configured sources
```bash
docjays list-sources
```

### Add a documentation source
```bash
docjays add-source --name my-docs --type git --url https://github.com/org/docs
```

### Sync all documentation
```bash
docjays sync
```

### Start MCP server for Claude
```bash
docjays serve
```

### Watch for changes (auto-sync)
```bash
docjays watch
```

### Check sync status
```bash
docjays status
```

## üìù Configuration

Edit `config.json` to customize DocJays behavior:

```json
{
  "sources": [],        // Documentation sources
  "mcp": {              // MCP server settings
    "enabled": true,
    "transport": "stdio"
  },
  "sync": {             // Auto-sync settings
    "auto": false,
    "interval": "1h"
  }
}
```

## ü§ñ Using with Claude

To use DocJays with Claude Desktop, add to your MCP settings:

```json
{
  "mcpServers": {
    "docjays": {
      "command": "docjays",
      "args": ["serve", "--stdio"]
    }
  }
}
```

Then restart Claude Desktop. Claude will now have access to all documentation in this folder!
